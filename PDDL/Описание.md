# 🛰️ SATELLITE MISSION PLANNER - Описание проекта

## 📌 Обзор

PDDL-планер для решения задачи спутника, который должен:
- Ротироваться в 8 направлениях (N, NE, E, SE, S, SW, W, NW)
- Фотографировать объекты в качестве HD или SD
- Отправлять фото на Землю (только когда указывает на N)
- Управлять ограниченной памятью (16 единиц) и слотами (max 2 фото)

Использует **Fast Downward** как основной планер + Python для парсинга и визуализации.

---

## ✅ Что уже сделано

### PDDL модель (`domain.pddl`)
- ✅ 4 типа предикатов: ориентация, видимость, состояние фото, ресурсы
- ✅ 6 действий: rotate-right, rotate-left, take-picture-hd, take-picture-sd, send-hd, send-sd
- ✅ Ограничения памяти и слотов
- ✅ Constants для 8 направлений

### Задачи (`problem_*.pddl`)
- ✅ `problem_easy.pddl` - 1 объект HD → 6 шагов
- ✅ `problem_medium.pddl` - 2 объекта HD → 12 шагов
- ✅ `problem_hard.pddl` - 3 объекта SD → 17 шагов
- ✅ `problem_extreme.pddl` - 4 объекта (HD+SD) → 20+ шагов

### Python инфраструктура

| Файл | Статус | Описание |
|------|--------|---------|
| `PE_IM_parser.py` | ✅ Работает | Парсит план из Fast Downward, извлекает имена действий |
| `PE_IM_animation.py` | ⚠️ Базовая | Печатает номер и название действия с паузой 1 сек |
| `PE_IM_planner_runner.py` | ✅ Полный | Запускает Fast Downward, тестирует 3 стратегии |
| `PE_IM_utils.py` | ✅ Полный | Константы (направления, cost памяти и т.д.) |
| `PE_IM_heuristics.py` | ✅ Полный | 5 эвристик для A* |
| `search.py` | ✅ Полный | Алгоритмы из AIMA (BFS, DFS, A*) |
